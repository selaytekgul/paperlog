# Paperlog production status — 2026-07-13

This record captures the verified state of Paperlog after the OAuth launch and Paper Picture integration. It intentionally contains no credentials, tokens, private database exports, or tester personal data.

## Current outcome

- Paperlog is publicly available at `https://paperlog.net` and `https://www.paperlog.net`.
- Both custom domains are active with active TLS/SSL and no reported domain error.
- The production health endpoint returns HTTP 200 with the D1 database connected.
- The Sites project is active and public.
- The current production application version is Sites version 17, built from source commit `490245c26286a81ba1bce9799bd925dfb6b5a1e4`.
- The GitHub `main` branch is clean and includes the later documentation-only commit `507aa1b`.

## Completed product work

- Research-paper discovery through OpenAlex, direct DOI, arXiv, OpenReview, and OpenAlex identifiers, plus a local fallback catalog.
- Paper pages, one-to-five-star ratings, reader notes, reading status, replies, helpful votes, lists, follows, notifications, and public profiles.
- Structured code and reproducibility experiences kept distinct from ordinary reader ratings.
- Reporting, contact, metadata corrections, author claims, moderation tools, rate limiting, and a private administrator console.
- Self-service account export and permanent deletion across Paperlog-owned account and activity records.
- Unified Google, GitHub, and managed ChatGPT sign-in. Google and GitHub use verified-email linking so matching identities return to one Paperlog profile.
- Protected OAuth credentials in the Sites environment. Only variable names are documented; secret values are not stored in Git.
- Public privacy, terms, guidelines, copyright, and contact pages.
- Paper Picture DOI integration with same-origin canonical redirects and an offline fallback catalog.

## Remaining work before wider promotion

1. Use a disposable, non-administrator production account to test sign-in, rating, note creation, editing, sign-out/sign-in persistence, account export, and permanent deletion. Do not use the owner's real account for the deletion test.
2. Download an administrator JSON backup and store it outside the repository in private encrypted storage.
3. Add a protected `OPENALEX_API_KEY` so normal server-side search does not depend on fallback behavior.
4. Run the first structured friend test from `docs/FRIEND_TEST_GUIDE.md`, fix any blocking issue, and then invite the remaining testers.
5. Before a broad launch, add automated backup/restore testing, stronger security headers and same-origin protections, privacy-conscious error monitoring, uptime monitoring, and qualified local legal/privacy review.

## Decision

The MVP is deployed and ready for a controlled small alpha. The next phase should prioritize validation, backup safety, search reliability, and observed tester problems rather than speculative feature development.
